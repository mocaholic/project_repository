    2. 코딩 가이드

    2-1. 코딩 기본사항- 접기

    빈 href 처리 개발요소 href 처리는 undefined 시킨다. 예) <a href='javascript:void(0);'>
    이미지 처리
    • 데이터 이미지가 아닌 경우 style sheet로 배경처리한다.
    데이터 Style sheet BO(운영)에서 제공하는 스타일은 element에 style 속성으로 처리한다. 예) <img src='smaple.jpg' alt='%게시물 제목%' style='width:200px; height:250px;'>
    기타 주의사항
    • id는 페이지 내 네비게이션 링크, 테이블 headers 제공 시 사용하는 경우를 제외하고 사용하지 않는다. 사용 id엔 style sheet 디자인 하지 않는다.
    • css 모든 파일은 kimchang.css에 import 하여 사용한다.
    • html5 문서에 xhtml 태그를 사용하지 않는다. 잘못 사용하는 예) <input type='text' />

    2-2. 네이밍규칙- 접기

    공통 잘 알려진 축약어를 제외하고 풀 네임을 사용한다.
    Class Design 스네이크표기법을 사용한다. ex) 버튼 : btn_, 이미지 : img_, 백그라운드 : bg_
    ID 카멜표기법 사용한다. 예) contentHeader
    document html 스네이크표기법 사용한다. 예) layout_main.html

    2-3. 웹접근성- 접기

    대체텍스트
    • 데이터 이미지의 경우 alt 속성으로 대체 텍스트 제공
    • 배경 이미지의 경우 글자 숨김처리 또는 title 속성으로 대체 텍스트 제공
    • QR코드 대체텍스트는 연결 URL을 포함시켜서 제공
    • 미디어 콘텐츠는 title 뿐만 아니고, 내용의 원고를 제공
    키보드 보장
    • blur() 내장함수 및 tabindex 속성 사용금지. 단, anchor가 아닌 element에 포커스 이동시에만 tabindex='0'을 사용한다.
    • 논리적인 element 코딩
    • shift + tab 기능 보장
    쉬운 네비게이션
    • 건너띄기 링크를 제공하고, '본문으로 이동'만 사용
    • 용도와 목적에 맞게 구체적인 링크 텍스트 제공 예) title='더보기' (x), title='공지사항 더보기' (o)
    이해할 수 있는 테이블 구조
    • caption은 th내용을 포함시킨다.
    • 2중 구조 이상의 th를 사용할 경우 반드시 headers를 제공한다.
    사용자 오류방지
    • 새창알림 title 속성의 value는 '새창 알림'으로 통일하여 제공한다.
    • 사용자 의도에 관계없이 운영자가 자동으로 데이터를 변경하지 않도록 한다.
    • 입력 폼 필드엔 label을 제공한다. label을 제공하기 곤란한 경우 title 속성으로 대체할 수 있다.
    사용자의 쉬운 이해
    • 모호한 안내 문장의 경우, 기획과 협업하여 구체적인 명시를 할 수 있도록 한다.
    • 컨텐츠의 명도대비 및 정보구분(데이터 뷰)의 경우, 디자인과 협업하여 사용자가 충분히 인식할 수 있는 수준으로 한다.
